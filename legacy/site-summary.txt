Couple Points 网站功能与页面总结
================================

整体功能
- 移动端优先的情侣积分 PWA，提供日常/每周打卡、特殊事件提交、奖励兑换与使用、留言板、审核中心等完整流程。
- 基于 Supabase Auth + Postgres + RLS 的登录与数据权限控制。
- 通过邀请码绑定情侣空间，限制两人组队，并区分“积分方(Earner)”与“审核方(Reviewer)”角色。

角色与权限
- 积分方(Earner)：完成打卡、提交特殊事件、发起奖励兑换与奖励使用、查看流水与留言、删除自己的留言。
- 审核方(Reviewer)：审核补签/特殊事件/兑换/使用申请，管理打卡规则与奖励，删除任意留言。

页面与路由
- `/`：入口加载页。根据登录状态与情侣绑定状态自动跳转到登录、绑定或首页。
- `/auth/login`：邮箱+密码登录。
- `/auth/register`：注册账号并填写昵称，创建用户资料后进入绑定流程。
- `/onboarding`：创建或加入情侣空间。创建时选择角色并生成邀请码；加入时输入邀请码；支持复制/分享邀请码与刷新绑定状态。
- `/home`：首页仪表盘，展示积分余额、今日/本周打卡进度、快捷入口、最近流水与留言；审核方还可看到待审事项预览。
- `/checkin`：打卡页面，支持每日/每周规则切换；积分方完成打卡并可发起“近 3 天补签”；审核方可创建/编辑/上下架规则。
- `/special/new`：积分方提交特殊事件（标题、描述、建议分值、发生日期），进入待审。
- `/ledger`：积分流水列表，按类型/状态/时间范围筛选，支持分页加载。
- `/ledger/:id`：流水详情页，展示规则/奖励关联信息；审核方可设置最终分值并通过/驳回。
- `/rewards`：奖励列表与管理。审核方可新增奖励与上下架；积分方可查看可兑换奖励。
- `/rewards/:id`：奖励详情。积分方发起兑换（余额不足时禁用）；审核方可编辑奖励信息与状态。
- `/rewards/uses`：奖励使用页，展示兑换/使用统计与剩余可用次数；积分方发起使用申请；审核方审核使用申请；提供使用记录。
- `/review`：审核中心，集中处理补签/特殊事件与兑换申请，支持快速通过/驳回与查看详情。
- `/review/redemptions/:id`：兑换审核详情页，展示申请人、奖励信息、审核结果与处理动作。
- `/messages`：留言板，支持发布留言、分页加载；审核方或留言作者可删除（软删除）。
- `/settings`：设置页，更新昵称与退出登录。
- `/more`：更多入口，聚合跳转到奖励、奖励使用、审核中心、留言板、设置。

关键业务逻辑（积分/审核/兑换/使用）
- 打卡规则分为“每日/每周”，每日打卡可补签（限制近 3 天，需审核），每周打卡直接生效。
- 特殊事件一律为待审，审核方可设置最终分值。
- 审核机制基于 transactions：通过时写入最终分值，驳回则标记拒绝。
- 兑换流程：发起兑换时生成 redemption 并创建预扣积分交易；审核通过后转为正式扣分，驳回则回退。
- 奖励使用流程：基于已兑换次数计算剩余可用数，积分方发起使用申请，审核方通过/驳回。
- 积分余额计算包含已通过交易，以及待审/已通过的兑换预扣。

技术与体验
- Next.js App Router + React 19；底部 TabBar 导航（首页/打卡/流水/更多）。
- PWA 配置（manifest、icons、standalone 显示、iOS 主屏支持）。
- 使用本地缓存加速首页仪表盘数据展示（localStorage）。

数据模型概览（与功能对应）
- profiles、couples、couple_members：用户昵称、情侣空间与角色。
- rules：打卡规则。
- transactions：打卡/特殊事件/兑换流水（含审核状态与分值）。
- rewards、redemptions、reward_usages：奖励、兑换申请、使用申请。
- messages：留言（支持软删除）。
